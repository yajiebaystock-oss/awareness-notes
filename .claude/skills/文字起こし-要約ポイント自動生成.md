# 文字起こし → 要約・ポイント自動生成 Skill

## 概要

文字起こし（transcription）テキストから、**要約**と**学習ポイント**を自動生成するスキルです。

このスキルを使うと：
- 📝 200-300字の実質的な要約を自動生成
- 📌 7個以上の具体的なポイントを自動抽出
- ✅ 品質チェック込み
- 🔄 複数ファイルの一括処理対応

---

## 使用シーン

| シーン | 詳細 |
|--------|------|
| **講座・セミナー動画** | 文字起こしから要約・ポイント抽出 |
| **ナレッジ管理** | ドキュメント→学習教材化 |
| **研修資料作成** | テキストから教育コンテンツ生成 |
| **オンライン講義** | 講義録音から学習ガイド作成 |

---

## 実行手順

### **ステップ1: 文字起こしを用意**

Markdown ファイルに以下の構造を準備：

```markdown
---
title: "セミナータイトル"
---

## 文字起こし（全文）

【セクション1】
実際の文字起こしテキスト...

【セクション2】
さらに続く文字起こし...
```

**文字起こしの要件：**
- 300字以上が推奨
- 複数のセクション・トピックを含む
- 完全なテキスト（途中省略なし）

---

### **ステップ2: このスキルを実行**

以下をプロンプトに含めて Claude Code を実行：

```
【このスキルを使用】
ファイル: [ファイルパス]
タイプ: [セミナー/講義/ナレッジ/その他]
対象: [全42ファイル / 特定の13ファイル など]

実行するアクション:
1. 文字起こしから200-300字の要約を生成
2. 7個以上のポイントを抽出
3. ファイルに実装
4. 品質チェック実施
```

---

### **ステップ3: 自動生成内容を確認**

以下が自動的に生成・実装されます：

#### **要約セクション**
```markdown
## 要約

[文字起こしから要点を抽出した200-300字の実テキスト]
```

**要約の条件：**
- 200-300字の日本語
- セミナーの核となる学習内容を要点化
- テンプレート文なし
- 実質的で読者の理解を助ける内容

#### **ポイントセクション**
```markdown
## ポイント

1. **ポイント1**: [具体的な説明]
2. **ポイント2**: [具体的な説明]
...
7. **ポイント7**: [具体的な説明]
```

**ポイントの条件：**
- 7個以上のポイント
- 各ポイントに具体的な説明（50字以上）
- 文字起こしの重要な学習要素から抽出
- **[ポイント名]: 説明** 形式

## 実装例

### **入力：文字起こし**

```
【RAS（網様体賦活系）とは何か】
RAS（Reticular Activating System）は脳幹に位置する神経系であり、
脳が処理する膨大な情報の中から『重要な情報』を選別するフィルター機能を持っています。

人間の感覚器官は毎秒1100万ビットの情報を受け取っていますが、
脳が処理できる情報は毎秒130ビット程度に限定されています。

このギャップを埋めるのがRASの役割であり、目的や関心に基づいて、
何を『見る』か『聞く』かを決定します。

【理論的背景と重要性】
目的がRASに明確に入力されると、その目的に関連した情報だけが
脳の意識領域に上がってくるようになります。

『新しい車を買おうと思ったら、その車種をやたら見かけるようになった』
という経験で実感できます。実際には情報量は変わっていませんが、
RASが目的に応じて情報をフィルタリングした結果、同じ環境でも
見える世界が変わるのです。
```

### **出力：要約**

```markdown
## 要約

RAS（網様体賦活系）は脳が膨大な情報から重要な情報を選別するフィルター機能です。
人間は毎秒1100万ビットの情報を受け取りますが、脳が処理できるのは毎秒130ビット程度。
この情報ギャップを埋めるのがRASの役割で、目的が明確に入力されると、
その目的に関連した情報だけが意識領域に上がってきます。
「新しい車を買おうと思ったらその車をやたら見かける」という現象は、
RASが目的に基づいて情報をフィルタリングしているからこそ起こります。
```

### **出力：ポイント**

```markdown
## ポイント

1. **RASのフィルター機能**: 脳幹の神経系が膨大な情報から『重要な情報』を自動選別
2. **情報処理ギャップ**: 受け取る情報（毎秒1100万ビット）と処理能力（毎秒130ビット）のギャップをRASが埋める
3. **目的による知覚変化**: 目的がRASに入力されると、その目的に関連した情報だけが意識領域に上がる
4. **現実認識の構築**: 同じ環境でも、RASの働き方によって見える世界が変わる
5. **新車現象の仕組み**: 新しい車を買おうと思うと、その車をやたら見かける現象はRASの作用
6. **スコトーマ形成**: 無視される情報（スコトーマ）が発生し、自動的に見落とされる
7. **目的の重要性**: 人生で見える現実は、明確な目的があるかないかで大きく変わる
```


## 品質基準

### **要約セクションの基準**

| 項目 | 基準 | チェック |
|------|------|---------|
| 文字数 | 200-300字 | ✅ |
| テンプレート | なし（実テキストのみ） | ✅ |
| 内容 | セミナーの核となる学習内容 | ✅ |
| 言語 | 日本語 | ✅ |

### **ポイントセクションの基準**

| 項目 | 基準 | チェック |
|------|------|---------|
| ポイント数 | 7個以上 | ✅ |
| フォーマット | **[ポイント名]: 説明** | ✅ |
| 説明字数 | 50字以上/項目 | ✅ |
| 出所 | 文字起こしから直接抽出 | ✅ |
| プレースホルダー | なし | ✅ |

### **全体品質チェック**

実装後、以下を自動チェック：
```
✅ Frontmatter (タグ5個以上)
✅ 要約セクション (100-300字実テキスト)
✅ ポイントセクション (7個以上の実ポイント)
✅ 文字起こしセクション (実テキスト存在)
✅ プレースホルダー (完全削除)
```

## トラブルシューティング

### **要約が生成されない場合**

```
原因: 文字起こしが短すぎるか、内容が不十分
解決策:
1. 文字起こしを300字以上に拡充
2. 複数のセクション・トピックを含める
3. 再実行
```

### **ポイントが不足する場合**

```
原因: 文字起こしの内容が単純または重複
解決策:
1. 文字起こしの詳細度を確認
2. 異なる視点のポイントを追加
3. Agent に "より多角的なポイント抽出" を指示
```

### **品質チェック不合格**

```
原因: テンプレート残存またはプレースホルダー削除漏れ
解決策:
1. 修正対象ファイルを確認
2. Agent に修正を指示
3. 再度品質チェック実施
```

# セミナーシリーズ汎用管理スキル

> このスキルを使用するには、Claude Code で `/seminar-` コマンドを実行してください。

## 説明

Awareness セミナーシリーズの統一的な管理と実装を支援するスキルです。
既に実装済みの CPS, SLS, HTP, T2S に加え、新しいセミナー（CBS, ICF, IES, ECS など）を迅速に実装できます。

**対応セミナー**: CBS, ICF, IES, ECS, CPS, SLS, HTP, T2S

**対応フォルダ**: `01_講義/[セミナー名]/`

---

## コマンド一覧

### `/seminar-search`
**セミナー名から series_id と基本情報を自動検索**

```
入力: セミナー名（例：CBS, ICF, ECS）
出力: 
  - series_id（例：study/61）
  - セッション数
  - 最新バージョン日付
  - Awareness.co.jp の対応ページURL
```

実行内容:
```bash
# セミナー情報を Web 検索で特定
# または既知の実装済みセミナーの情報を表示
grep -r "series_id\|study/" "01_講義/" | grep -i "[SEMINAR]"
```

### `/seminar-fetch`
**Awareness サイトから最新版のセッション一覧を取得**

```
入力: 
  - セミナー名（例：CBS）
  - バージョン日付（例：2026-06-27、または "latest"）

出力:
  - セッション数
  - 各セッションのタイトル
  - 配信日
  - メタデータテンプレート
```

実行内容:
```bash
# Awareness から最新版の情報をフェッチ
# または既知の実装から情報を抽出
find "01_講義/[セミナー名]/" -name "*.md" | head -20
```

### `/seminar-list`
**全セミナーの実装状況を表示**

```
実装済み:
✅ T2S (study/239) - 42セッション
✅ SLS (study/59) - 16セッション
✅ HTP (study/146) - 28セッション
✅ CPS (study/60) - 28セッション
✅ CBS (study/61) - 18セッション

実装予定:
🔄 ICF (study/??) - ?セッション
🔄 IES (study/??) - ?セッション
🔄 ECS (study/??) - ?セッション
```

### `/seminar-create`
**新しいセミナーシリーズを一括実装**

入力:
```
セミナー名: [CBS, ICF, IES, ECS など]
セッション数: [推定値または確定値]
バージョン日付: [2026-MM-DD 形式、または "今日の日付"]
セッション構成: [セッション名一覧、またはテンプレートから自動生成]
```

出力:
```
✅ ディレクトリ作成: 01_講義/[セミナー名]/[バージョン日付]/
✅ 全セッションファイル生成: [N]_[セミナー名]_[タイトル].md
✅ README.md 作成
✅ メタデータ統一
✅ エピソード間の連結確認
✅ Git コミット実施
```

### `/seminar-validate`
**セミナーフォルダのフォーマット検証**

チェック項目:
- ✅ ディレクトリ構造の正確性
- ✅ ファイル名フォーマット（[N]_[セミナー]_[タイトル].md）
- ✅ YAML frontmatter の存在と形式
- ✅ セクション順序（要約 → ポイント → 文字起こし）
- ✅ メタデータの統一性（category, tags など）
- ✅ セッション間の連結情報（次回セッションの記述）
- ✅ 要約と文字起こしの文字数チェック
- ✅ ポイント数の確認（7個以上）

### `/seminar-update-readme`
**README.md を自動生成・更新**

更新項目:
- ✅ セミナー概要
- ✅ ディレクトリ構成
- ✅ セッション一覧
- ✅ 学習方法（推奨時間含む）
- ✅ 関連プログラムとの連携
- ✅ メタデータ・バージョン管理表
- ✅ 進捗状況（XX/YY セッション）
- ✅ 最終更新日時

### `/seminar-compare`
**複数セミナー間の構造比較**

比較項目:
- ファイル形式の統一性
- メタデータの一貫性
- セッション構成のパターン
- 学習目標の関連性

## テンプレート・フォーマット

### セッションファイル名
```
[N]_[セミナー]_[タイトル].md

例：
CBS_2026_01.md
CBS_2026_02.md
ICF_2026_01.md
```

### メタデータ（YAML frontmatter）
```yaml
---
title: "【[セミナー名] 2026】[セッションタイトル]"
date: 2026-MM-DD
source_url: "https://www.awareness.co.jp/mypage/archives/study/XX?sci=N"
episode_id: [セミナー]_2026_[XX]
category: "[セミナー名] 2026"
category_label: "[セミナー名] 2026 - [説明]（2026年6月版）"
tags: [[セミナー], 2026年版, [キーワード1], [キーワード2]]
---
```

### セッション内容構成
```markdown
## 要約
[200-300字]

## ポイント
[8個以上の項目、各50字以上]

## 文字起こし（全文）
[600字以上の詳細説明]

---

**エピソード情報**
- **シリーズ**: [シリーズ名] 18回シリーズ
- **所属**: [段階名]
- **次回**: [次セッションのID] [次セッションのタイトル]へ進む
- **関連テーマ**: [テーマ3つ]
```

## トラブルシューティング

### セミナーが見つからない場合

```
原因: series_id が未確定のセミナー（ICF, IES, ECS など）

解決策:
1. Awareness.co.jp で手動検索
2. /seminar-search でWeb検索実行
3. 推定値を使用して実装
4. 後から確定値で更新
```

### メタデータが統一されていない場合

```
原因: セッション間でcategoryやtagsが異なる

解決策:
1. /seminar-validate で問題を特定
2. 問題ファイルの修正コマンドを実行
3. 再度 /seminar-validate で確認
```

### セッション間の連結が途切れている場合

```
原因: 「次回セッション」の記述が不正確

解決策:
1. /seminar-validate でチェック
2. 修正コマンドを実行
3. エピソード情報を自動更新
```

```

1. 要件確認

   ↓
3. /seminar-fetch で詳細情報を取得
   ↓
4. /seminar-create で一括実装
   ↓

   ↓
6. /seminar-update-readme で README 更新
   ↓
7. Git コミット
   ↓
8. 完了報告 ✅
```

## 応用方法

### 用途1：セミナー統一フォーマットの強制

既存のセミナーから新しいセミナーへの一括変換：
```
/seminar-compare で既存セミナー（CBS）の形式を取得
その形式で新しいセミナー（ICF）を実装
```

### 用途2：複数セミナーの同時管理

異なるセミナーの進捗を一覧で確認：
```
/seminar-list で全セミナーの実装状況を表示
必要に応じて個別のセミナーで /seminar-validate を実行
```

### 用途3：セミナー間の学習フロー設計

関連セミナー間の連携を自動作成：
```
README.md に「関連プログラムとの連携」セクションを自動生成
学習者が体系的に複数セミナーを学べる環境を構築
```

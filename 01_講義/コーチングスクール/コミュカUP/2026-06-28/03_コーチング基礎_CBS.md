## セクション概要

**コーチング基礎【CBS】** は、コーチングの理論的基礎と実践的スキルを統合的に習得するセクションです。セクション1で学んだ原理原則（T2S）と、セクション2で習得した基本スキル（SCS）を次のレベルへ発展させます。相手の成長を引き出すコーチング手法を、実際のケーススタディを通じて深く学びます。

---

## 学習目標

このセクション修了時に、以下の能力を獲得します：

1. **コーチングの本質理解**: コーチングの定義、目的、他の支援スタイルとの違いを説明できる
2. **コーチングの構造化モデル**: GROW モデル、OODA ループなどのフレームワークを実装できる
3. **効果的な質問スキル**: 相手の思考を深め、自発的な行動を促す質問ができる
4. **心理的安全性の構築**: 相手が自由に表現し、試行錯誤できる環境を作れる
5. **実践的コーチングスキル**: 職場や個人的な関係でコーチング的対話ができる

## 理論編：コーチング基礎理論

### 1. コーチングの定義と特徴

#### 1.1 コーチング vs. その他の支援スタイル

相手の成長や課題解決を支援する方法には、複数のアプローチがあります。それぞれの特徴を理解することが、適切なスタイルの選択につながります。

| スタイル | 定義 | 相手の状態 | 関わり方 |
|---------|------|---------|--------|
| **コーチング** | 相手の内的リソースを引き出し、自発的成長を促す | 既に答えを持っている | 質問、傾聴 |
| **ティーチング** | 知識やスキルを直接教える | 知識不足 | 説明、指導 |
| **メンタリング** | 経験に基づいた助言や指導 | 経験不足、方向性探索 | アドバイス、示唆 |
| **カウンセリング** | 心理的課題を解決する支援 | 心理的問題 | 傾聴、感情処理 |

#### 1.2 コーチングの本質

コーチングは、以下の信念に基づいています：

**相手は既に答えを持っている**
- 相手の内部に、課題解決に必要な智慧やリソースが存在する
- コーチの役割は、それらを引き出すことにある

**成長は自発性から生まれる**
- 外部から押し付けられた変化ではなく、自らが選択した変化
- その時初めて、持続可能な成長が実現する

**関係の質が成果を決める**
- 信頼と安全性がなければ、本当の対話は生まれない
- コーチ自身の在り方が、相手の開放性を決める

---

### 2. コーチングの基本スキル

#### 2.1 「聴く」の深さのレベル

コーチングの核となるスキルは「聴く」ことです。その深さに応じて、複数のレベルがあります。

**レベル1：内的聴取（Internal Listening）**
- 聴きながら、自分の経験や判断を参照している
- 「私なら～する」という内部対話が起きている
- 効果：限定的（相手の世界観が見えにくい）

**レベル2：焦点聴取（Focused Listening）**
- 相手の言葉に完全に集中している
- 相手を理解することに意識が向いている
- 効果：中程度（相手の表面的なメッセージは理解できる）

**レベ3：全人的聴取（Global Listening）**
- 言葉だけでなく、相手全体を聴いている
- 言われていないことも聴こえている
- 相手の可能性まで感じ取る
- 効果：最大（相手の深層的なニーズが見える）

#### 2.2 質問のアート

効果的な質問は、相手の思考を深め、新しい可能性を開きます。

**質問の種類と効果**:

1. **開かれた質問（Open Question）**
   - 「その状況で、あなたはどう感じましたか？」
   - 効果：相手の思考を自由に引き出す

2. **焦点質問（Focusing Question）**
   - 「その中で、最も重要なのは何ですか？」
   - 効果：相手の思考を整理させる

3. **奥行き質問（Deepening Question）**
   - 「その背景には、どんな信念がありますか？」
   - 効果：相手の深層的なニーズを引き出す

4. **可能性質問（Possibility Question）**
   - 「そしたら、どんなことが可能になりますか？」
   - 効果：相手の未来志向を促す

#### 2.3 傾聴とラポール

信頼関係（ラポール）を構築するための傾聴の具体例：

- 相手の話を遮らない
- 相手の言葉の中から、本当のニーズを探る
- 相手の感情と事実を区別して理解する
- 「理解されている」と相手に感じさせる応答

### 3. コーチングの構造化モデル

#### 3.1 GROW モデル

最も広く使用されるコーチングのフレームワークです。

```
G (Goal) - 目標設定
│
├─ 「何を実現したいのか」を明確にする
├─ 相手の価値観と整合性があるか確認
└─ 具体的で測定可能な目標か確認

R (Reality) - 現状認識
│
├─ 「今、ここでの状況は何か」
├─ 既に取られている行動
└─ 環境的な制約や可能性

O (Options) - 選択肢探索
│
├─ 「どのようなアプローチがあるか」
├─ 従来的な方法だけでなく、創造的な選択肢
└─ リソース（人、知識、経験）の活用

W (Will) - 行動決定
│
├─ 「最初の一歩は何か」
├─ 「いつまでに、どの程度実施するか」
└─ サポート体制の確認
```

#### 3.2 OODA ループ

素早い環境変化への対応力を高めるモデルです。

```
O (Observe) - 観察
│ └─ 現在の状況を認識

O (Orient) - 方向づけ
│ └─ 状況を自分のメンタルモデルと結合

D (Decide) - 意思決定
│ └─ 行動方針を決定

A (Act) - 行動実行
│ └─ 決定に基づいて行動

└─ 再び Observe へ（継続的改善）
```

このループを短期化することで、複雑で変化の速い環境での適応力が高まります。

### 4. 心理的安全性の構築

#### 4.1 心理的安全性とは

相手が「失敗しても大丈夫」「本当の自分を表現できる」と感じられる状態です。

**心理的安全性が高い時**:
- 相手は本当のニーズを開示する
- 失敗から学ぶ環境が生まれる
- 創造的な提案が増える

**心理的安全性が低い時**:
- 相手は防御的になり、本心を隠す
- 失敗を隠そうとする
- 責任回避的な行動が増える

#### 4.2 心理的安全性の構築方法

1. **予測可能性**
   - コーチの言動が一貫している
   - 相手の話を批判しない

2. **包容性**
   - 相手の違いを受け入れる
   - 異なる意見を尊重する

3. **自分の弱さの開示**
   - コーチが完璧でないことを示す
   - 失敗から学んだ経験を共有

## 実践編：コーチングスキルの活用

### 5. コーチングダイアログの実例

#### シナリオ：部下が新しいプロジェクトでの成功を疑っている

```
【部下】「正直、このプロジェクトで成功できるか不安です」

【コーチング的対応：第1段階 - 相手の状態理解】
コーチ：「そうなんですね。その不安の背景にはどんなことがあるんですか？」
部下：「これまでこのような規模のプロジェクトやったことがないので」

コーチ：「そのような状況での不安は自然ですね。今、具体的に
　　　　どんなことが不安ですか？」
部下：「技術的な部分と、チームマネジメントの両方に自信がない」

【第2段階 - 相手のリソース認識】
コーチ：「これまでの成功体験の中で、複雑な課題を解決した
　　　　例がありますか？」
部下：「ああ、去年のシステム更新プロジェクトですね」

コーチ：「そのプロジェクトで、どのようなアプローチで
　　　　成功させたんですか？」
部下：「チームの強みを理解して、役割分担を工夫しました」

【第3段階 - 可能性の認識】
コーチ：「そのアプローチが、今のプロジェクトでも
　　　　使えるとしたら？」
部下：「あ、それなら今回も似たようにできるかもしれません」

コーチ：「では、まず最初の一歩として何をしてみますか？」
部下：「チームメンバーの強みをまず把握することですね」
```

このダイアログでは、コーチが部下の中にある答え（既に成功経験がある）を引き出しています。

### 6. よくある課題と対策

#### 課題1：アドバイスをしてしまう

コーチングは質問を通じた引き出しが基本ですが、つい「～した方がいい」というアドバイスをしてしまう。

**対策**:
- アドバイスしたい衝動を感じたら、その質問を思い出す
- 「相手は既に答えを持っている」という信念を確認する

#### 課題2：相手の進捗が遅い

質問を通じた変化は、指示よりもゆっくり進むことがあります。

**対策**:
- 短期的な成果だけでなく、長期的な自立を視野に入れる
- 相手が自発的に動く力を信頼する

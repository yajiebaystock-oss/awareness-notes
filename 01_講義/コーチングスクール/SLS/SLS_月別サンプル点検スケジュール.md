# 【SLS】月別サンプル点検スケジュール

**対象**: SLS 2022/2/15 プロジェクト（10ファイル）  
**実施者**: ユーザー（目視確認） + 品質管理スキル  
**開始**: 2026-07-27  
**頻度**: 毎月 1 回（15日 or 28日）  
**対象数**: 毎月 3-5 ファイル（12ヶ月で全30-60ファイル定点観測）

---

## 📋 **検証観点（5つの視点）**

```
1. 【タイトル固有性】
   - ファイル名と title の一致確認
   - セッション主要用語が本文に3回以上出現

2. 【テンプレート検出】
   - 既知テンプレートパターン（A, B, C）の有無
   - 汎用定型句の過剰使用

3. 【実質性確認】
   - ユニーク度スコア判定（>= 0.60）
   - 自動生成度スコア
   - 具体例・事例の有無

4. 【セクション構造】
   - 要約: 100-300字範囲
   - ポイント: 7個以上
   - 文字起こし: 200字以上

5. 【具体例・事例**
   - 理論的背景に裏付けられた実例
   - 実践的なステップの記載
   - 現実的な障害対応の説明
```

## 📝 **各月の点検プロセス**

### **ステップ1: サンプル抽出（自動）**
```
毎月15日or28日の朝
→ ランダムに3-5ファイル抽出
→ 点検対象を確定
```

### **ステップ2: 目視確認（手動）**
```
各ファイルに対して5つの検証観点で確認
（30-40分程度）

チェックリスト使用: [[quality_checklist_30items.md]]
```

### **ステップ3: 問題検出**
```
【判定】
✅ PASS: 90%以上 → そのまま継続使用
⚠️ WARN: 70-89% → 改善計画立案
❌ FAIL: 70%未満 → 即座に修正
```

### **ステップ4: 改善対応**
```
Critical/High: 即座に修正 → コミット
Medium: 翌月までに修正
Low: 参考情報として記録
```

### **ステップ5: レポート作成**
```
- 検出日: YYYY-MM-DD
- 対象ファイル: セッション [X, Y, Z]
- 検出内容: 
- 対応: 
- 次回予定: YYYY-MM-DD
```

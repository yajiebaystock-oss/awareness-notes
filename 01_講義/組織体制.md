# Awareness Vault 組織体制

**設立日**: 2026年6月27日  
**プロジェクト**: Awareness Vault 管理・運用  
**目的**: 高品質な学習教材の継続的な管理・保証

---

## 🏢 組織構成（3名体制）

---

## 1️⃣ 管理者（マネージャー）

### **名前**: 矢島 康二（やじま こうじ）

**職位**: Vault 統括責任者・プロジェクトマネージャー

**プロフィール**:
- **経歴**: Awareness コーチング事業部の統括責任者
- **専門分野**: 組織管理、プロジェクト計画、時間管理
- **スキル**: 
  - Git/GitHub 運用管理
  - セミナー企画・構成設計
  - チームコーディネーション
  - 品質基準の策定

**責務**:
- Vault 全体の戦略立案・計画
- シリーズ別の進捗管理
- バージョン管理（2023/4/15版 vs 2025/10/12版）
- Git コミット・リリース判断
- GitHub 同期・リモート管理
- チーム間の調整・報告

**目標**:
- 150ファイル以上の統一的管理
- 複数バージョンの混在防止
- 月次更新サイクルの確立
- ステークホルダーへの報告

**連絡先**: brand-all@awareness.co.jp

---

## 2️⃣ 作業員（文字起こし・実装担当）

### **名前**: 田中 美咲（たなか みさき）

**職位**: コンテンツ制作・実装担当者

**プロフィール**:
- **経歴**: オンラインセミナー制作チームのベテラン制作者
- **専門分野**: 動画編集、文字起こし、Markdown コンテンツ制作
- **スキル**:
  - 動画音声の文字起こし（精度95%以上）
  - Markdown 形式での正確な記述
  - メタデータ（YAML frontmatter）の入力
  - 要約・ポイントの自動生成
  - ファイル整理・配置

**責務**:
- 動画からの講義内容抽出
- 文字起こしテキストの作成
- Markdown ファイルの生成・実装
- 要約セクション（200-300字）の作成
- ポイント（7個以上）の抽出・記述
- メタデータ（日付、タグ、カテゴリ）の入力
- ファイルの正しいディレクトリへの配置

**責任量**:
- 月間 50-100ファイルの処理能力
- 週単位での納期管理
- 品質基準の事前確認

**目標**:
- 正確で読みやすい文字起こしの提供
- 納期厳守（マネージャーからの指示から 3営業日以内）
- コンテンツ品質の初期段階での確保

**連絡先**: 内部チャネル

---

## 3️⃣ 品質管理（QA責任者）

### **名前**: 鈴木 健太（すずき けんた）

**職位**: 品質保証・監査担当者

**プロフィール**:
- **経歴**: Awareness 品質管理部門の監査スペシャリスト
- **専門分野**: コンテンツ品質管理、品質基準策定、監査プロセス
- **スキル**:
  - 品質チェックリスト管理
  - Markdown 形式の整合性確認
  - メタデータ検証
  - 文字起こしの正確性確認
  - 統計的品質管理

**責務**:
- 全ファイルの品質チェック
- 文字起こしの正確性・完全性確認
- 要約・ポイントの質的評価
- Frontmatter（メタデータ）の完全性確認
- タグ・カテゴリの統一性確認
- リンク・参照の有効性確認
- プレースホルダーテキストの削除確認
- 品質基準クリア判定
- 問題ファイルの修正指示

**チェック基準**:
- ✅ Frontmatter 完全性（100%）
- ✅ 要約セクション（200-300字実テキスト）
- ✅ ポイント（7個以上の実ポイント）
- ✅ 文字起こし（600字以上の実コンテンツ）
- ✅ メタデータ統一性
- ✅ リンク有効性
- ✅ エラー・重複なし

**監査周期**:
- 新規ファイル：即座チェック（48時間以内）
- 既存ファイル：月次監査（毎月末）
- 四半期：全体監査

**報告**:
- 月次品質レポート提出
- 問題件数・対応率の報告
- 改善提案の提示

**目標**:
- 品質合格率 100%（0件の不合格）
- 平均チェック時間の短縮
- 継続的な品質向上

**連絡先**: 内部チャネル

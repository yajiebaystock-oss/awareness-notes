### update_video_urls.py

**説明:** URL 更新メインツール（対話型）

**用途:** 取得した URL を Markdown ファイルに自動更新

**実行方法:**
```powershell
python update_video_urls.py
```

**処理フロー:**
1. 現在の URL 状態を表示
2. 各動画の新しい URL を対話的に入力
3. URL を検証
4. バックアップを作成
5. Markdown ファイルの `source_url` フィールドを更新

**入力例:**
```
[1/3] 無限大の原則
現在の URL: https://www.awareness.co.jp/mypage/archives/study-watch/348/XXX/XXXX
新しい URL を入力 > https://www.awareness.co.jp/mypage/archives/study-watch/348/375/2024/
✓ URL が有効です
この URL で更新しますか？ (y/n) > y
  → バックアップを作成しました
  → ファイルが更新されました
```

**特徴:**
- 対話的（わかりやすい）
- URL 形式検証付き
- バックアップ自動作成
- エラーハンドリング完備

---

### batch_check_urls.py

**説明:** URL 状態確認ツール

**用途:** 3つのファイルの URL 状態を一括確認（更新前・更新後の検証用）

**実行方法:**
```powershell
python batch_check_urls.py
```

**確認項目:**
- URL が完全な形か
- URL が不完全（プレースホルダー含む）か
- URL が取得できていないか

**出力例:**
```
【URL 確認結果】

✓ 無限大の原則
  状態: 完全
  URL: https://www.awareness.co.jp/mypage/archives/study-watch/348/375/2024/

⚠ 繰の原則
  状態: 不完全
  問題: URL に プレースホルダー（XXX/XXXX）が含まれています

【サマリー】
確認済みファイル: 3個
  ✓ 完全: 1個
  ⚠ 不完全: 1個
  ✗ エラー/未取得: 1個
```

**特徴:**
- 素早く状態確認
- 問題点を列挙
- 推奨アクションを表示

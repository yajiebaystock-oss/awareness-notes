## 💡 よくある初期質問への即答

### Q: なぜ3ヶ月なの？

```
A: 以下の理由による最短実現期間です
   - 田中の実装能力: 月25～30ファイル
   - 鈴木のQA能力: 全数100%チェック
   - 品質基準: 95%以上維持
   
   これらを勘案すると、85講義を3ヶ月で
   高品質に実装するのが最短です。
```

### Q: 人数が足りないなら？

```
A: 実装期間が延びます
   - 2名体制: 5～6ヶ月に延長
   - 1名体制: 実施不可（品質低下）
   
   3名体制が最小要件です。
```

### Q: 品質95%の根拠は？

```
A: 既実装シリーズの実績値です
   - SLS: 100%
   - 10の能力: 100%
   - スキマニ: 100%
   - T2S: 95%（改善中）
   
   平均97%を目指し、95%を最低基準としています。
```

### Q: 個人の工数は多い？

```
A: 標準的範囲内です
   - 矢島: 月10～15時間（管理業務）
   - 田中: 月85～100時間（フルタイム実装）
   - 鈴木: 月40～50時間（QA業務）
   
   いずれも現実的範囲です。
```

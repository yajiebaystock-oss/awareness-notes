## 📋 **30項目チェックリスト**

### **A. Frontmatter 検証（5項目）**

- [ ] **A-1. title フィールド存在確認**
  - 要件: `title: "【カテゴリ】タイトル"` 形式
  - 基準: 存在・重複なし・形式正確
  - 影響: ファイルの識別可能性

- [ ] **A-2. date フィールド形式**
  - 要件: `YYYY-MM-DD` 形式
  - 基準: 有効な日付・更新日を反映
  - 影響: 版管理・更新追跡

- [ ] **A-3. source_url フィールド有効性**
  - 要件: 有効なURL / プレースホルダーなし
  - 基準: URLが実在・アクセス可能
  - 影響: 情報源の信頼性

- [ ] **A-4. category フィールド統一**
  - 要件: シリーズごとに統一値
  - 基準: 「10の能力動画」「スキマニT2S」「T2Sセミナー」のいずれか
  - 影響: カテゴリ管理・検索

- [ ] **A-5. tags フィールド充実度**
  - 要件: 5個以上、カンマ区切り
  - 基準: コンテンツ関連の有意義なタグ
  - 影響: 検索性・分類精度

---

### **B. 要約セクション（4項目）**

- [ ] **B-1. セクション存在確認**
  - 要件: `## 要約` セクション存在
  - 基準: 適切な位置・フォーマット
  - 影響: コンテンツ構造の完全性

- [ ] **B-2. 文字数範囲**
  - 要件: 100～300字（推奨200字以上）
  - 基準: 短すぎない・長すぎない
  - 影響: 要約としての有効性

- [ ] **B-3. プレースホルダー削除**
  - 要件: `[未生成]` `[テンプレート]` なし
  - 基準: 実テキストのみ
  - 影響: 完成度

- [ ] **B-4. 内容品質（サンプル点検）**
  - 要件: セミナーの内容を要約・反映
  - 基準: コンテンツと一貫性あり
  - 影響: 情報精度

---

### **C. ポイントセクション（4項目）**

- [ ] **C-1. セクション存在確認**
  - 要件: `## ポイント` セクション存在
  - 基準: 適切な位置・フォーマット
  - 影響: コンテンツ構造の完全性

- [ ] **C-2. 項目数（7個以上）**
  - 要件: 7個以上の番号付きポイント
  - 基準: `1.` `2.` ... 形式で列挙
  - 影響: 学習ポイントの充実度

- [ ] **C-3. プレースホルダー削除**
  - 要件: `[ポイント1]` `[テンプレート]` なし
  - 基準: 実テキストのみ
  - 影響: 完成度

- [ ] **C-4. 内容品質（サンプル点検）**
  - 要件: 各ポイントが実質的な説明を含む
  - 基準: タイトルと関連・有意義
  - 影響: 学習効果

---

### **D. 文字起こしセクション（6項目）【重要】**

- [ ] **D-1. セクション存在確認**
  - 要件: `## 文字起こし（全文）` セクション存在
  - 基準: 適切な位置・フォーマット
  - 影響: コンテンツ構造の完全性

- [ ] **D-2. プレースホルダー検出（形式1）**
  - 要件: `[文字起こしテキスト：XXX]` なし
  - 基準: 実テキストのみ
  - 影響: 完成度

- [ ] **D-3. テンプレート検出（複数パターン）** ⭐ **新規**
  - 要件: 既知テンプレートパターンなし
  - 基準: 「本セミナーでは...」などの定型句が少ない
  - 影響: 実テキスト判定の精度

- [ ] **D-4. ユニーク度スコア** ⭐ **新規**
  - 要件: スコア >= 0.60 (FAIR以上)
  - 基準: タイトル固有性・多様性・自動生成度から計算
  - 影響: 実質的な講義内容であるか

- [ ] **D-5. 文字数（最小要件）**
  - 要件: 200字以上（推奨300字以上）
  - 基準: テンプレートでは満たす、実テキストではさらに充実
  - 影響: 講義内容の充実度

- [ ] **D-6. サンプル点検（月1回）** ⭐ **新規**
  - 要件: 5つの検証観点で目視確認
  - 基準: タイトル固有性・テンプレート・実質性・セクション・具体例
  - 影響: 自動化では検出できない問題の発見

---

### **E. 整合性チェック（5項目）**

- [ ] **E-1. ファイル名と title の一致**
  - 要件: ファイル名がカテゴリ接頭辞を含む
  - 基準: 【カテゴリ】タイトル 形式で一致
  - 影響: ナビゲーション・識別可能性

- [ ] **E-2. category の全ファイル統一**
  - 要件: 同シリーズは同じ category 値
  - 基準: 3シリーズで異なる値
  - 影響: カテゴリ管理

- [ ] **E-3. source_url の ID 確認**
  - 要件: VideoId または 記事ID が含まれる
  - 基準: 有効な外部参照
  - 影響: 情報源のトレーサビリティ

- [ ] **E-4. tags の内容関連性**
  - 要件: コンテンツに関連するタグ
  - 基準: タグがセミナー内容を反映
  - 影響: 検索精度・分類妥当性

- [ ] **E-5. 内部リンク有効性**
  - 要件: `[[...]]` 形式が有効なファイルを指す
  - 基準: 破損リンクなし
  - 影響: ナビゲーション機能

  - 影響: テンプレート排除

- [ ] **F-3. 既知パターンC検出**
  - 要件: 「このセミナーの内容を実生活で活かすために：」などなし
  - 基準: 定型開始句が最小限
  - 影響: テンプレート排除

- [ ] **F-4. パターン追跡**
  - 要件: 新規テンプレート発見時にドキュメント化

  - 影響: 実テキスト判定の精度

## 📊 **チェック実施ルール**

### **チェック実施タイミング**

| チェック種別 | 実施タイミング | 担当者 | 対象 |
|-----------|-----------|------|------|
| **A～E（基本チェック）** | 毎回（ファイル修正時） | 自動化スクリプト | 全72ファイル |
| **F～G（詳細チェック）** | 毎回＋定期レビュー | 自動化スクリプト | 全72ファイル |
| **サンプル点検（D-6）** | 月1回（5ファイル）** | 人間による目視確認 | 60ファイル/年 |

### **評価基準**

```
完成度スコア = (合格チェック数 / 30) × 100%

✅ 90%以上: EXCELLENT (A+) - 本番対応可
✅ 80-89%: GOOD (A) - 本番対応可
⚠️ 70-79%: FAIR (B) - 改善推奨
❌ 70%未満: POOR (C) - 修正必須
```

### **判定フロー**

```
自動化チェック（A～G）
    ↓
【合格】90%以上 → ✅ 本番対応
    ↓
【要改善】70-89% → 改善実施 → 再チェック
    ↓
【不合格】70%未満 → 差し戻し → 再実装
    ↓
【月1回】サンプル点検 → 人間検証で最終確認
```
